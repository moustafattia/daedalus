Hermes Relay Architecture

A durable orchestration runtime around a workflow policy brain: wrapper decides semantic workflow truth, Relay owns leases, actions, retries, failures, service supervision, and operator tooling.

Design intent

  • • Separate workflow semantics from durable orchestration
  • • Keep wrapper as policy brain
  • • Let Relay own leases, queues, failures, and supervision
  • • Preserve explicit actor handoffs across long-lived lanes

State model

  • • SQLite = canonical runtime truth now
  • • JSONL = append-only history forever
  • • lane memo/state = durable handoff artifacts
  • • GitHub remains external source of issue/PR truth

Long-term vision

  • • 24/7 agentic SDLC lanes
  • • explicit coder / reviewer / orchestrator roles
  • • durable repair loops and retries
  • • humans passive by default, in the loop when needed

Hermes Relay • workflow brain + durable orchestrator • operator-visible, failure-aware, lane-oriented automation
